### Conventional Commits

#### Commit message는 <타입: 설명> 형태로 작성하며, 타입은 아래와 같습니다.
    - feat (새로운 기능 추가)
    - fix (버그 수정)
    - docs (문서 수정)
    - style (코드 스타일 변경)
    - refactor (리팩토링)
    - test (테스트 코드 추가/수정)
    - chore (빌드, 패키지 등 환경 관련 작업)

#### 작성 예시는 아래와 같습니다.
    - feat: allow provided config object to extend other configs
    - docs: correct spelling of CHANGELOG
    - fix: prevent racing of requests

### 배포 과정
    1. 코드 수정
    2. 변경사항 커밋
    3. 커밋 내용 푸시
    4. firebase deploy
